k8s_install_ingress_with_traefik

  • kubernetes

前言

Kubernetes对外暴露服务可以有三种方式：

  • LoadBlancer

    这种方式主要是通过向云平台申请创建一个负载均衡器来向外暴露服务端口

  • NodePort

    通过在集群上的每个node端口来暴露服务

  • Ingress

    通过使用nginx, traefik等开源反向代理负载均衡器实现对外暴露服务端口

    ​

这里我们主要介绍的是Ingress的方式，Ingress包含三个组件：

  • 反向代理负载均衡器

    反向代理负载均衡器其实就是nginx，traefik；在集群中反向代理负载均衡器可以自由部署，可以使用 Replication Controller、Deployment、DaemonSet 等方式

  • Ingress Controller

    它实质上可以看作是一个监视器，它不断的与Api Server打交道，实时感知后端service, pod的变化，比如新增和减少pod, service增加与减少等，当发现有变化时，它再与下面要介绍的ingress通信生成配置，然后更新反向代理负载均衡器，刷新配置，从而达到负载均衡的目的。

  • Ingress

    它简单理解就是规则的定义；比如说某个域名对应某个service，也就是某个域名的请求进来时转发给某个service; 这个规则将与 Ingress Controller 结合，然后 Ingress Controller 将其动态写入到负载均衡器配置中，从而实现整体的服务发现和负载均衡

    ​

那么这里使用nginx和traefik有什么不同呢？

traefik本身会与api server打交道，来监听服务的变化，从而替代了原生的Ingress Controller,实现配置热更新，不像nginx还需要进行配置的reload操作，traefik可以动态的实现服务发现。
